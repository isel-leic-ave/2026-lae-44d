## Introduction

In this lab, you will work at two levels of abstraction: Kotlin source code and
JVM bytecode. You will start by reading a sequence of bytecode instructions and
deriving the Kotlin code that produces the same behavior.

In the second part, you will move from inspection to generation by using the
Class-File API to create a class file programmatically. In the final part, you
will extend that work to generate a small stateful class, including a field, a
constructor, and an instance method that depends on that field.

Throughout the lab, use bytecode inspection tools such as `javap` to compare
compiled code with generated code and to avoid mistakes in the emitted class
structure.

## Part 2: Metaprogramming

Implement a Kotlin function `buildBar(): KFunction<*>` that uses the Class-File
API to generate a class file that contains a method with the same bytecode as
above. It should return a reference to the generated method.

Make a unit test that calls the resulting method from `buildBar()` with some
sample inputs and checks that the result of invoking that method is the same as
the result of calling the equivalent Kotlin function you wrote in Part 1.

## Part 3: Generating a Class with State

Consider the following Kotlin class:

```kotlin
class RetryPolicy(private val maxRetries: Int) {
    fun canRetry(attempt: Int): Boolean {
        return attempt < maxRetries
    }
}
```

Implement a Kotlin function `buildRetryPolicy(): KClass<*>` that uses the
Class-File API to generate a class file equivalent to `RetryPolicy`. The
function should return the `KClass` representing the generated class.

Before writing `buildRetryPolicy()`, compile the Kotlin version above and inspect
its bytecode with `javap -c -p RetryPolicy`. Use that output as a guide so that
you do not make mistakes when generating the constructor, the field, and the
`canRetry` method.

Also write a unit test that:

* creates an instance of the generated `RetryPolicy` class with the value `3`
* invokes `canRetry` with a few sample values
* checks that the observed results match the expected behavior

For example, with `maxRetries = 3`:

* `canRetry(0)` should return `true`
* `canRetry(2)` should return `true`
* `canRetry(3)` should return `false`
